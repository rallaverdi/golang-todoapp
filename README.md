# Docker compose сервисы

## todoapp-service
- (environment:) чтобы не хранить чувствительные данные в файле, они вынесены в переменные POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB
- (volumes:) если контейнер умрет или рестартанет мы не потеряем данные , потому что данные будут пробрасываться на машину из докер сети
- ...|... слева указывается директория куда будут пробрасываться данные из контейнера, а справа какая директория будет пробрасываться ИЗ контнейра (использовать лучше полные пути)
- (ports:) можно было пробросить порт из докер сети на локальную машину, то это рисковано, если например на продовом сервере мы деплоим новое приложение и тогда возникнет конфликт портов, поэтому в сервисе todoapp-postgres мы не выбросили порт сервиса наружу
- (restart:always) если сервер упадет с ошибкой, то будет перезапущен , уменьшает downtime postgres

## todoapp-postgres-migrate
- библиотека для миграций

## port-forwarder
- говорим что будет слушать на своем порту порт 5432 из сервиса todoapp-postgres
- (command:) говорим что будет слушать на своем порту порт 5432 из сервиса todoapp-postgres
- (ports:) если злоумышленник узнает айпи сервера сможет получить доступ к порту а значит и доступ к БД, поэтому указываем явно что доступ будет только с localhost


# Makefile

## команды в файле:
- все команды заканчивающиеся на -windows это по сути костыли или попытка реализации этих команд не в рамках unix системы
- команды без нее - оригинальные
- команда которая помогла накатить миграцию вручную сохранена в переменной "MANUAL_MIGRATION_QUERY" в .env

## аргументы для "migrate-create":
- docker compose run отрабатывает 1 раз , up - поднимается и живет
- create команда чтобы создать файлы миграции
- ext sql название расширений файла
- dir папка где искать файлы для миграций
- seq название для миграций migrate-create seq=some_value
